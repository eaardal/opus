This is a work task management app with a heavy emphasis on organizing tasks in order of required completion. The main workspace of the app is a canvas that we place the tasks on and connect them with connections/arrows. By connecting Task A with Task B (task A points at task B), we say that Task A must be completed before Task B can start. When doing this for all tasks, we build a graph of tasks and the order in which they must be completed. This is inspired by the concept of the Critical Path Method in project management.

## Technologies

- **Wails** - The project started as a desktop app using Wails and Go to load a TypeScript frontend, but we also deploy the frontend as a standalone website which has become the default way of using the app. Wails will maybe be removed at a later point.
- **React and TypeScript** - The app is mainly a React app using custom made UI elements on a HTML canvas.
- **Firestore** - Real time database as backend

## Commands

### From project root

[Mise](https://mise.jdx.dev/) is the primary task runner for the project as a whole.

- `wails dev` - Run the project locally
- `mise run generate` - Generate Wails JS bindings from Go types
- `mise run generate:changelog` - Reads CHANGELOG.md and generates changelog.ts from it, that is used to show a changelog in the app
- `mise run build` - Build the desktop app with Wails for the current OS platform
- `mise run build:mac` - Build the desktop app with Wails for MacOS
- `mise run build:linux` - Build the desktop app with Wails for Linux
- `mise run build:windows` - Build the desktop app with Wails for Windows
- `mise run package` - Build and package the desktop app for all OSes
- `mise run package:mac` - Build and package the desktop app for MacOS
- `mise run package:linux` - Build and package the desktop app for Linux
- `mise run package:windows` - Build and package the desktop app for Windows

### From ./frontend

NPM is the primary frontend task runner

- `npm run dev` - Run the frontend locally
- `npm run build` - Build the frontend
- `npm run test` - Run all tests
- `npm run test:once` - Run all tests once
- `npm run test:coverage` - Run all tests with coverage
- `npm run format` - Format all frontend code
- `npm run lint` - Lint all frontend code
- `npm run verify` - Convenience script for checking that build, format, lint and tests are OK
